小结

1.堆叠注入
注入时，用‘;’把多个语句分开，这样注入时就可以同时运行多个语句。和union操作符不同的是，union使用的语句类型是有限的。
详细：https://www.cnblogs.com/lcamry/p/5762905.html

2.增删查改语句

1)增：
增数据：insert into 表名（例1，例2，…….）values (v1,v2,…….);
// insert into student(id,name,grade) values (1,’zhangshan’,98);
增数据库：create database 数据库名;
增除表：create table 表名;
2)删：
删数据:
delete from 表名 [where id=1];
//delete from users where id=1;
删数据库：drop database 数据库名;
删除表：drop table 表名;
删除表中的列:alter table 表名 drop column 列名;
3)改：update 表名 set 字段=’值’ [where 条件]；
// update users set username= ‘ABC’ where id=’1’;
4)查：select */字段列表 from 表名 [where 条件];
// select */username from users [where id=’1’];

和上面差不多，只是是整型注入，然后注意一下构造的语句中不要有'即可。

宽字节注入

宽字节： GB2312、GBK、GB18030、BIG5、Shift_JIS等这些都是常说的宽字节，实际上只有两字节。
宽字节带来的安全问题主要是ASCII字符(一字节)的现象，即将两个ascii字符误认为是一个宽字节字符。
中文、韩文、日文等均存在宽字节，英文默认都是一个字节。在使用PHP连接MySQL的时候，
当设置“set  character_set_client = gbk”时会导致一个编码转换的问题。

例子： id= 1’处理1\’进行编码1%5c%27带入sql后id=\’and XXXX此时无法完成注入
id=1%df’处理1%df\’进行编码1%df%5c%27带入sql后id=1運’and XXX 此时存在宽字节注入漏洞

参考文章：https://blog.csdn.net/helloc0de/article/details/76180190
https://blog.csdn.net/heiseweiye/article/details/82723478
推荐解码网站：http://www.mytju.com/classcode/tools/urldecode_gb2312.asp
